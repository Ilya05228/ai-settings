# Выбор языка ответа

Используй этот skill, когда язык ответа явно не задан.

## Правило

- если пользователь явно указал язык ответа, следуй этому
- иначе выбери язык по содержательной части самой задачи
- если более `80%` задачи написано на одном языке, отвечай на нем

## Что игнорировать

Не учитывай при выборе языка:

- slash-команды
- названия skills, плагинов и инструментов
- системные и служебные вставки
- URL, названия пакетов, репозиториев и команд
- код, трассировки, пути к файлам и конфигурационные ключи
- отдельные английские технические термины внутри неанглоязычной задачи

Сначала мысленно очисти запрос до нормальной формулировки задачи, потом выбирай язык.

## Если порог не достигнут

- ориентируйся на язык основной постановки задачи
- если задача реально сбалансирована и язык важен, задай короткий уточняющий вопрос

## Правило для кода

- код, идентификаторы, имена файлов, функций, классов и API всегда пиши на английском
- комментарии в коде, если они нужны, пиши на выбранном языке ответа
- если комментарии не нужны, не добавляй их

## Проверка
- язык не был задан явно
- язык выбран по самой задаче, а не по шуму
- код остался на английском
- комментарии, если они есть, совпадают с языком ответа
