## programming-best-practices

Локальный плагин с архитектурными практиками программирования.

Что входит:
- skill `ddd`
- skill `clean-architecture`
- skill `domain-events`
- skill `java` (каталог `skills/java-best-practices/`; соглашения по Java; триггерится при любой работе с Java-кодом)
- стартовые evals для всех skills

Как использовать:
- подключай `ddd`, когда хочешь явно применять Domain-Driven Design, богатую доменную модель, `Value Object`, агрегаты и типизированные идентификаторы
- подключай `clean-architecture`, когда хочешь явно применять слои, порты и адаптеры, границы зависимостей и отделение бизнес-логики от инфраструктуры
- подключай `domain-events`, когда нужно отдельно проработать смысл, форму, буферизацию и публикацию доменных событий
- подключай skill `java`, когда в задаче есть Java (описание skill настроено так, чтобы его выбирали при работе с `.java` и Java-кодом)
- можно подключать несколько skills сразу, если задача требует и сильной доменной модели, и чистых архитектурных границ, и отдельной проработки событий

Основные правила:
- архитектурные skills (`ddd`, `clean-architecture`, `domain-events`) не используют `alwaysApply` — их подключают по смыслу задачи
- skill `java` рассчитан на частое срабатывание при Java через описание в frontmatter, а не через `alwaysApply`
- если skill подключен, его правила считаются обязательными для текущей задачи

Установка в Cursor:

```bash
./commands install --dest ./.cursor --plugins programming-best-practices
```

Переустановка поверх уже установленных скиллов/агентов того же имени (перезапись):

```bash
./commands install --dest ./.cursor --plugins programming-best-practices --force
```
