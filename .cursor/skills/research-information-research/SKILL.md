# Поиск информации (MCP: research stack)

## Инструменты в этом стеке

| MCP-сервер | Когда выбирать |
|------------|----------------|
| **context7** | Нужны **актуальные доки библиотеки/фреймворка** (API, версии, сигнатуры, best practices пакета). Не заменяет общий веб-поиск. Требует `CONTEXT7_API_KEY`. |
| **duckduckgo** | **Открытый веб**: новости, статьи, блоги, страницы продуктов, всё что индексируется как публичный сайт. Удобно для «свежей» общей информации без своего поискового API. |
| **grep** ([mcp.grep.app](https://mcp.grep.app)) | **Код на GitHub**: найти реализацию, usage паттерна, импорты, типичные вызовы API по репозиториям. Не для чтения произвольного URL как браузер. |
| **playwright** | **После** того как есть URL (или список): открыть страницу, дождаться JS, снять снимок доступности, кликнуть, заполнить форму, пройти по разделам. Для тяжёлого SPA и антибота гарантий нет. |

## Алгоритм (два этапа)

### Этап 1 — откуда искать

1. Сформулируй цель: *док пакета* / *общий веб* / *пример кода в репо* / *содержимое конкретной страницы*.
2. Выбери **один главный** канал (при необходимости добавь второй):
   - доки npm/pip/crate и т.д. → **context7**;
   - «что пишут в интернете про X» → **duckduckgo**;
   - «как другие вызывают `foo` в TypeScript» → **grep**;
   - уже знаешь точный URL и нужен DOM/форма/многостраничный флоу → можно сразу **playwright**, если не нужен предварительный поиск.
3. Выполни поиск, **сохрани список источников**: название, URL, зачем этот линк.

### Этап 2 — получить информацию

4. Для каждого релевантного URL из списка: открой через **playwright** (навигация, снимок, извлечение нужного текста/структуры).
5. Если страница **простая и статичная**, иногда достаточно отдельного **fetch**-MCP (если подключён вне этого плагина) — быстрее и дешевле; иначе **playwright**.
6. Сведи вывод: факты с указанием **источника и даты**, что осталось неясным.

## Правила

- **Не дублируй** три поисковика подряд с одним и тем же запросом — сначала выбери канал по таблице.
- **Context7** не для «любой темы в мире» — только для **документации библиотек** в поддерживаемом индексе.
- **grep** — про **кодовую базу GitHub**, не про PDF и не про закрытые репо.
- **Playwright** — после того как **есть куда идти**; для массового сбора ссылок сначала duckduckgo или grep.

## Краткая блок-схема

```text
Задача
  → нужны доки пакета?     → context7 → [URLs] → playwright (при необходимости)
  → нужен веб/статьи?      → duckduckgo → [URLs] → playwright
  → нужны примеры кода?    → grep → [URLs] → playwright (GitHub / страницы)
  → уже есть URL?          → playwright (и при необходимости уточнить запросом выше)
```

## Объявление в начале задачи

Кратко скажи пользователю: какой канал выбран и почему (одно предложение), затем выполняй этапы 1–2.
